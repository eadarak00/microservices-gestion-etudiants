# Contribuer au projet Microservices Gestion Etudiants

Ce document fournit des directives pour contribuer à ce projet. Il est destiné à l'équipe de développement.

## Pour commencer

1.  **Cloner le dépôt** :
    ```bash
    git clone https://github.com/eadarak00/microservices-gestion-etudiants.git
    ```
2.  **Créer une branche** pour votre fonctionnalité ou correction de bug :
    ```bash
    git checkout -b feature/ma-nouvelle-fonctionnalite
    ```

## Processus de Pull Request

1.  Assurez-vous que le code compile et que les tests passent avant de soumettre.
2.  Mettez à jour le fichier `README.md` si nécessaire (nouveaux ports, variables d'environnement, etc.).
3.  Respectez le versionnage sémantique (SemVer) pour les numéros de version.
4.  Une Pull Request nécessite la validation d'au moins un autre développeur avant d'être fusionnée.

## Signaler des Bugs

-   **Utilisez un titre clair et descriptif** pour le problème.
-   **Décrivez les étapes exactes** pour reproduire le problème.
-   **Fournissez des exemples précis**.
-   **Décrivez le comportement attendu** par rapport au comportement observé.

## Normes de Codage

-   **Java** : Suivez les conventions de nommage et les bonnes pratiques Java standard.
-   **Commits** : Rédigez des messages de commit clairs et concis.
